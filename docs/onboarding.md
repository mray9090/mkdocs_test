# Team Onboarding Guide

This page outlines the basic resources and access needed for new Infrastructure team members.

---

## Required Access

New team members should request access to the following:

- Active Directory admin groups (read-only initially)
- Azure subscription (Reader role)
- VMware vCenter (Read-only)
- GitHub organization access
- Monitoring platform dashboards

> Access should follow least-privilege and be expanded as needed.

---

## Tools We Use

| Tool | Purpose |
|-----|--------|
| GitHub | Source control & documentation |
| Ansible AAP | Configuration management |
| Ivanti Security Controls | Windows patching |
| LogicMonitor | Monitoring |

---

## First Week Checklist

- [ ] Review VM build documentation  
- [ ] Walk through patching workflow  
- [ ] Review CIS baseline overview  
- [ ] Gain read-only production access  

---

## Notes

All production changes must be documented and reviewed.  
If something isn’t documented, assume it is **not supported**.
